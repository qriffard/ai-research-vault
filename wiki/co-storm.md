# Co-STORM

**Co-STORM** (Collaborative STORM) extends [[storm]] from a fully automated
writing system into an interactive information-seeking assistant where humans
observe and participate in multi-agent discourse. From [[stanford-oval]];
lead authors [[yucheng-jiang]] and [[yijia-shao]]; advised by [[monica-lam]].

## Motivation

Traditional search engines and RAG chatbots passively react to user queries,
inducing echo chamber effects and placing high cognitive load on users who may
lack the prior knowledge to formulate good questions. STORM produces useful
articles but offers no user interaction. Co-STORM bridges this gap by letting
users learn from **observing** agent conversations and **steering** them — akin
to a student listening to experts discuss a topic and occasionally jumping in.

The key target is surfacing **unknown unknowns** — information users don't know
they're missing — through serendipitous discovery during discourse.

## Architecture

Three participant types engage in turn-based collaborative discourse $\mathcal{D} = \{u_1, u_2, \ldots, u_n\}$:

### Roundtable Experts

Multiple LM agents with distinct perspectives (e.g., "AI Expert", "Geneticist",
"Molecular Biology Expert" for the topic AlphaFold3). Each expert's turn:
1. LM chooses an **utterance intent**: Original Question, Information Request,
   Potential Answer, or Further Details.
2. If answering: generate a search query → retrieve → cite response.
3. Polish the utterance to be conversational and engaging.

Perspectives are generated by giving the LM background on the topic and asking
it to suggest diverse expert roles — same approach as
[[perspective-guided-question-asking]] in STORM.

### Moderator

A non-expert agent that injects new directions. Generates questions from
**unused retrieved information** reranked by:

$$\text{score} = \cos(\mathbf{i}, \mathbf{t})^\alpha \cdot (1 - \cos(\mathbf{i}, \mathbf{q}))^{1-\alpha}$$

This prioritizes information relevant to the topic $t$ but dissimilar to the
current question $q$ — steering discourse toward unexplored territory.

Ablation shows removing the moderator hurts more than reducing the number of
experts, confirming its role as the primary driver of serendipitous discovery.

### Human User

Can observe passively (auto-steering mode runs experts + moderator) or intervene
at any turn to inject a question or opinion. User utterances trigger an expert
list refresh so subsequent discourse reflects the new direction.

## Dynamic Mind Map

A tree-structured concept hierarchy $\mathcal{M} = (\mathcal{C}, \mathcal{E})$
that organizes collected information in real time:

- **Insert** — places new information under the most semantically similar
  concept, with LM arbitration.
- **Expand** — when a concept accumulates >$K$ pieces of information, the LM
  generates subtopics and redistributes.
- Bottom-up cleanup removes empty/singleton nodes.

The mind map serves as shared knowledge between user and system, reducing
cognitive load. It also doubles as the report outline.

## Discourse Protocol

- **Mixed-initiative**: user can take a turn at any point; otherwise auto-steering.
- **Turn management**: experts cycle sequentially; after $L$ consecutive
  answer/detail turns, the moderator intervenes.
- Upon conclusion, the system generates a full-length cited report section by
  section using the mind map as skeleton.

## Evaluation

### WildSeek dataset

A new benchmark constructed from real information-seeking records on the public
STORM website. Contains topics + user goals across multiple domains.

### Automatic evaluation

Co-STORM significantly outperforms baselines on both report quality and
discourse quality:
- Report: higher Depth and Novelty vs STORM+QA and RAG Chatbot.
- Discourse: question-answering turns score higher on Consistency and Engagement.
- Question quality improves most from the moderator role.

### Human evaluation (N=20, IRB-approved)

| Comparison | Preference for Co-STORM |
|---|---|
| vs Search Engine | **70%** |
| vs RAG Chatbot | **78%** |

Participants found Co-STORM:
- Surfaces broader and deeper information (50–67% rate strictly higher breadth).
- Provides more serendipitous discovery with less mental effort.
- 89% of moderator utterances rated as "steering toward new and interesting direction."
- Mind map accurately tracked discourse 71% of the time.

## Limitations

- Does not adapt to user's prior knowledge level.
- Users want more control (utterance length, perspective management).
- Single-language only (English); multilingual extension non-trivial.
- Higher latency than RAG chatbot due to intent decisions + mind map updates.

## API

```python
costorm_runner.warm_start()         # build shared conceptual space

```
